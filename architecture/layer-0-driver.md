# Layer 0 — Driver

The only layer that talks to hardware. GPIO registers, UART peripherals, ADC channels, timer hardware — all of that lives here. Nothing above Layer 0 ever calls `gpio_set_level()` or reads an ADC channel directly.

---

## tmc2209_driver

The TMC2209 is a stepper motor driver with two notable features: StealthChop2 (silent operation via sinusoidal current shaping) and StallGuard2 (sensorless end-stop detection via back-EMF monitoring).

### UART interface

The TMC2209 uses half-duplex single-wire UART at 115200 baud on GPIO18 (`PDN_UART`). The datagram format:

```
SYNC (0x05) | SLAVE_ADDR (0x00) | REGISTER | DATA[4] | CRC-8
```

CRC polynomial: `0x07`. No external TMC2209 library is used — the driver writes register values directly using raw UART datagrams. This keeps the dependency list clean and makes it straightforward to understand exactly what's being sent to the chip.

### Key registers

| Register | Address | Purpose |
|----------|---------|---------|
| GCONF | 0x00 | Bit 0: enable UART mode (pdn_disable=1) |
| IHOLD_IRUN | 0x10 | Run and hold current (0–31 scale) |
| TCOOLTHRS | 0x14 | Minimum speed for StallGuard2 to activate |
| CHOPCONF | 0x6C | Microstepping resolution (16 microsteps) |
| SGTHRS | 0x40 | Stall detection threshold (set to 50) |
| DRVSTATUS | 0x6F | Read-only: stall, open load, overtemp flags |
| PWMCONF | 0x70 | StealthChop2 configuration |

### Current setting

Run current is set via the IRUN field in IHOLD_IRUN. The formula:

```
cs = (rms_ma × 4980) / (R_SENSE_mOhm × 1000) - 1
```

With R_SENSE = 110 mΩ (R1 on the PCB near the TMC2209), a 800 mA RMS target gives cs ≈ 35. Clamped to 0–31. Hold current is set lower to reduce idle heat.

### Step pulse generation

Step pulses go on GPIO5. These are generated by the ESP32-S3 GPTimer hardware — not software delays. The ISR callback runs at each alarm, toggles GPIO5, and adjusts the timer period to implement a trapezoidal acceleration ramp. This keeps pulse timing consistent regardless of what the RTOS is doing in the background.

### StallGuard2 and end-stop detection

The DIAG pin on the TMC2209 connects to GPIO16. When the motor stalls — curtain hits a wall, rod is blocked — back-EMF drops and the TMC2209 raises DIAG HIGH. The ESP32-S3 has a rising-edge interrupt configured on GPIO16. The ISR posts a `MOTOR_EVT_STALL` event to the curtain FSM queue.

`TCOOLTHRS` must be set correctly. StallGuard2 only activates above a minimum speed threshold. Below that threshold — roughly 5% of rated speed — back-EMF is too small to be reliable. Physical limit switches on AUX1 (GPIO14) and AUX2 (GPIO13) cover this edge case.

## button_handler

Three buttons: OPEN (GPIO35), STOP (GPIO36), CLOSE (GPIO37). All active LOW.

- 50 ms debounce implemented in software
- Long press detection (>3 s) on Button 3 triggers factory reset: clears NVS `wifi_creds` namespace and reboots into BLE provisioning mode
- Button events are posted to a FreeRTOS queue consumed by the curtain FSM — same queue used by BLE and MQTT

## led_indicator

Four LEDs driven via GPIO. `led_set_curtain_state()` is the only public function. The LED driver maintains its own pattern state machine; callers never touch GPIO directly.

| State | LED pattern |
|-------|-------------|
| OPENING / CLOSING | All LEDs pulse together |
| Fully OPEN | LED3 (GPIO18) solid GREEN |
| Fully CLOSED | LED4 (GPIO19) solid GREEN |
| ERROR / STALL | LED1 (GPIO10) fast blink RED |
| No Wi-Fi | LED2 (GPIO12) slow blink RED |
| MQTT connected | LED2 (GPIO12) solid GREEN |

## limit_sensor

Monitors AUX1 (GPIO14) and AUX2 (GPIO13) as hardware backup end-stops. These are normally-open microswitches. When activated, the driver posts `MOTOR_EVT_LIMIT_HIT` to the FSM queue.

Used during slow calibration runs and as a safety net if StallGuard2 fails to detect a stall at very low speed.

## ntc / vbus monitoring

Two ADC channels read periodically:

- **GPIO7 (NTC):** 10 kΩ NTC thermistor near the TMC2209. Temperature is calculated using the Steinhart-Hart beta equation (β = 3950, R₀ = 10 kΩ at 25°C). If temperature exceeds 70°C, the driver posts an overheat event and stops the motor.

- **GPIO4 (VBUS_SENSE):** Monitors the 12 V USB-PD rail. If voltage drops below 10 V (charger removed or PD negotiation failed), the driver flags a power fault.
